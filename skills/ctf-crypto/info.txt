# RTAF Crypto Anti-Trap Playbook (Execution Mode)

เวอร์ชันนี้คือโหมดใช้งานจริงระหว่างแข่ง: สั้น, deterministic, และกันหลุมพราง

## 0) Non-Negotiable Rules

1. Assume ทุกอย่างอาจเป็น decoy จนกว่าจะพิสูจน์ได้
2. ห้ามเชื่อค่าตัวแรกที่เห็น (`n,e,c`, key, endpoint, hint)
3. ทุก candidate flag ต้องผ่าน provenance check ก่อน submit
4. ถ้า error เดิมซ้ำ 3 ครั้ง ให้ pivot ทันที
5. output flag ให้ทันทีเมื่อยืนยันแล้ว:

[FLAG] <raw_flag>
[CLEAN] <sanitized_flag>
[METHOD] <one-line extraction command>

## 1) Trap Taxonomy (Quick)

- Context confusion: ค่าจริงไม่ใช่ค่าที่ประกาศบนสุด
- Red herring: เหมือน small-e/Wiener ได้ แต่จริง ๆ ใส่ padding/เงื่อนไขเพิ่ม
- Fake flag injection: มี flag ปลอมให้ regex ติดก่อน
- Multi-stage gate: ต้องผ่าน state/captcha/time window ก่อนถึงข้อมูลจริง
- Encoding maze: base64/hex/reverse/xor ซ้อนชั้น
- Dependency trap: โค้ดใช้เวอร์ชัน/tool เฉพาะ

## 2) Anti-Trap Execution Pipeline

Step A: Artifact sanity
- ตรวจขนาดไฟล์ก่อนอ่านทั้งก้อน
- ดึง candidate ทั้งหมดของ `n,e,c` (ไม่ใช่ตัวแรกตัวเดียว)
- เช็ก validity: `c < n`, `gcd(e,n)=1`, bit-length สมเหตุผล

Step B: Attack routing (evidence-first)
- route จากหลักฐานรวม ไม่ใช่ pattern เดียว (เช่นเห็น `e=3` อย่างเดียวห้ามสรุป)
- ถ้ามีคำว่า partial bits/small roots -> Coppersmith/Sage ก่อน

Step C: Decode + verify
- decode chain แบบมี score/provenance
- wrapper flag มาก่อน raw hash token
- หากเจอหลาย candidate ให้เลือกตัวที่มีหลักฐาน chain ครบสุด

Step D: Submission guard
- กัน fake flag: verify จาก source + method reproducible 1 คำสั่ง
- บันทึกเหตุผล reject ของ decoy ทุกตัว

## 3) Programming Mode (ตามที่ขอ)

ใช้เมื่อโจทย์เป็น script-heavy / custom logic / implementation trap

### 3.1 Programming Anti-Trap Checklist

1. Parse, don’t trust comments
- comment อาจหลอกว่า AES แต่โค้ดจริงเป็น custom cipher

2. Build minimal oracle tests
- เขียน unit probe สั้น ๆ ตรวจสมบัติจริง (bijection? padding? randomness?)

3. Deterministic harness ก่อน brute force
- fix seed / timeout / retries / logging เพื่อแยก bug กับ trap

4. Dual-path validation
- อย่างน้อย 2 วิธีต้องยืนยันคำตอบเดียวกัน (math check + decode check)

5. Fail-fast policy
- timeout endpoint: ลด rate/parallel
- error ซ้ำ 3 ครั้ง: เปลี่ยนสมมติฐาน ไม่วนที่เดิม

### 3.2 Programming Template (One-Screen)

```python
# 1) Collect all candidates (no first-value trust)
candidates = collect_candidates(source)

# 2) Sanity filter
valid = [x for x in candidates if is_valid_rsa_tuple(x)]

# 3) Score & route by evidence
plan = route_attack(valid, hints, behavior)

# 4) Run deterministic solver
result = run_solver(plan, timeout=5, retries=2)

# 5) Decode chain with provenance
decoded = decode_with_trace(result.bytes)

# 6) Verify candidate flags
flag = select_verified_flag(decoded.candidates)

# 7) Emit competition format
emit_flag(flag, method=decoded.method)
```

### 3.3 What to avoid in Programming Mode

- เขียนสคริปต์ยาวก่อนพิสูจน์ hypothesis
- brute force ทั้ง space โดยไม่มี pruning
- submit ทันทีเมื่อ regex ติดครั้งแรก
- ปน stdout log กับ JSON machine output

## 4) Fast Decision Matrix

- เจอ `e=3` + message เล็ก + no padding evidence -> small-e
- เจอ “same N, different e” -> common modulus
- เจอ “partial bits / small roots” -> coppersmith (Sage)
- เจอ “anomalous / ECC order = p” -> smart attack (Sage)
- ไม่มี hint ชัด + multilayer data -> decode-chain with scoring

## 5) Definition of Done (Anti-Trap)

ถือว่าจบเมื่อครบทุกข้อ:
- มี evidence chain จาก input -> transform -> output
- มี one-line reproduce method
- flag candidate ผ่าน provenance validation
- ไม่มี error เดิมซ้ำโดยไม่ pivot

### **1. Context Confusion Defense**

#### **File: `helpers/context_analyzer.py`**

```python
#!/usr/bin/env python3
"""
Context Analyzer - Detect and extract REAL parameters from obfuscated code
"""
import re
import ast
from typing import Dict, List, Any

class ContextAnalyzer:
    def __init__(self, source_code: str):
        self.source = source_code
        self.candidates = {
            'n': [],
            'e': [],
            'c': [],
            'p': [],
            'q': [],
            'd': []
        }
    
    def extract_all_assignments(self):
        """Extract ALL variable assignments, not just first occurrence"""
        
        # Method 1: Regex extraction
        for var in ['n', 'e', 'c', 'p', 'q', 'd']:
            # Match: var = <value>
            pattern = rf'{var}\s*=\s*([0-9a-fxA-FX]+)'
            matches = re.findall(pattern, self.source)
            self.candidates[var].extend(matches)
            
            # Match: var = int("...", base)
            pattern = rf'{var}\s*=\s*int\(["\']([^"\']+)["\']\s*,?\s*(\d+)?\)'
            matches = re.findall(pattern, self.source)
            for value, base in matches:
                base = int(base) if base else 10
                self.candidates[var].append(str(int(value, base)))
        
        # Method 2: AST parsing (more reliable)
        try:
            tree = ast.parse(self.source)
            for node in ast.walk(tree):
                if isinstance(node, ast.Assign):
                    for target in node.targets:
                        if isinstance(target, ast.Name):
                            var_name = target.id
                            if var_name in self.candidates:
                                # Extract value
                                value = self._extract_value(node.value)
                                if value:
                                    self.candidates[var_name].append(value)
        except:
            pass
        
        return self.candidates
    
    def _extract_value(self, node):
        """Extract value from AST node"""
        if isinstance(node, ast.Constant):
            return str(node.value)
        elif isinstance(node, ast.Num):
            return str(node.n)
        elif isinstance(node, ast.Call):
            # Handle int("0x...", 16)
            if isinstance(node.func, ast.Name) and node.func.id == 'int':
                if node.args:
                    arg = node.args[0]
                    if isinstance(arg, ast.Constant):
                        base = 10
                        if len(node.args) > 1:
                            base = node.args[1].value
                        return str(int(arg.value, base))
        return None
    
    def rank_candidates(self, var: str) -> List[str]:
        """
        Rank candidates by likelihood of being real value
        Heuristics:
        1. Larger values more likely for n, c
        2. Common values (65537) likely for e
        3. Values in comments less likely to be decoys
        4. Last assignment usually real (overwrite pattern)
        """
        candidates = self.candidates.get(var, [])
        if not candidates:
            return []
        
        # Convert to integers
        int_candidates = []
        for c in candidates:
            try:
                int_candidates.append(int(c, 0))
            except:
                pass
        
        if not int_candidates:
            return []
        
        # Ranking rules
        if var == 'n':
            # Larger N more likely (RSA modulus should be big)
            int_candidates.sort(reverse=True)
        elif var == 'e':
            # Common exponents first (65537, 3, 17)
            common_e = [65537, 3, 17, 5, 257]
            int_candidates.sort(key=lambda x: (x not in common_e, x))
        elif var == 'c':
            # Larger ciphertext more likely
            int_candidates.sort(reverse=True)
        
        return [str(c) for c in int_candidates]
    
    def get_best_params(self) -> Dict[str, Any]:
        """Get most likely real parameters"""
        self.extract_all_assignments()
        
        result = {}
        for var in ['n', 'e', 'c', 'p', 'q', 'd']:
            ranked = self.rank_candidates(var)
            if ranked:
                result[var] = ranked[0]  # Best candidate
                result[f'{var}_all'] = ranked  # All candidates
        
        return result

def analyze_challenge_file(filepath: str) -> Dict[str, Any]:
    """Analyze challenge file and extract real parameters"""
    with open(filepath, 'r') as f:
        source = f.read()
    
    analyzer = ContextAnalyzer(source)
    params = analyzer.get_best_params()
    
    # Additional checks
    if 'n' in params and 'e' in params:
        n = int(params['n'], 0)
        e = int(params['e'], 0)
        
        # Sanity checks
        if n.bit_length() < 512:
            print(f"[!] WARNING: N is suspiciously small ({n.bit_length()} bits)")
            print(f"[!] Trying next candidate...")
            if 'n_all' in params and len(params['n_all']) > 1:
                params['n'] = params['n_all'][1]
    
    return params

# Example usage
if __name__ == '__main__':
    import sys
    result = analyze_challenge_file(sys.argv[1])
    print("Extracted parameters:")
    for k, v in result.items():
        if not k.endswith('_all'):
            print(f"{k} = {v}")
```

### **2. Multi-Stage Challenge Bypass**

#### **File: `helpers/stage_detector.py`**

```python
#!/usr/bin/env python3
"""
Multi-Stage Challenge Detector and Orchestrator
Automatically detects stages and coordinates execution
"""
import re
import json
from typing import List, Dict, Any
from pathlib import Path

class StageDetector:
    def __init__(self, challenge_dir: Path):
        self.challenge_dir = Path(challenge_dir)
        self.stages = []
    
    def detect_stages(self) -> List[Dict[str, Any]]:
        """
        Detect challenge stages from:
        1. File naming (stage1.py, stage2.py)
        2. Comments in code (# Stage 1:, # Stage 2:)
        3. Network endpoints (/stage/1, /stage/2)
        4. README instructions
        """
        
        # Method 1: File-based stages
        stage_files = sorted(self.challenge_dir.glob('stage*.py'))
        stage_files.extend(sorted(self.challenge_dir.glob('step*.py')))
        
        for f in stage_files:
            stage_num = int(re.search(r'(\d+)', f.stem).group(1))
            self.stages.append({
                'number': stage_num,
                'type': 'file',
                'path': f,
                'requires_human': self._check_human_required(f)
            })
        
        # Method 2: Code comment stages
        for py_file in self.challenge_dir.glob('*.py'):
            with open(py_file) as f:
                content = f.read()
                stage_comments = re.findall(r'#\s*Stage\s*(\d+):\s*(.+)', content, re.IGNORECASE)
                for stage_num, description in stage_comments:
                    self.stages.append({
                        'number': int(stage_num),
                        'type': 'comment',
                        'description': description.strip(),
                        'file': py_file,
                        'requires_human': 'physical' in description.lower() or 'location' in description.lower()
                    })
        
        # Method 3: Network-based stages
        if (self.challenge_dir / 'server.py').exists():
            with open(self.challenge_dir / 'server.py') as f:
                content = f.read()
                endpoints = re.findall(r'@app\.route\(["\']([^"\']+)["\']\)', content)
                for endpoint in endpoints:
```
